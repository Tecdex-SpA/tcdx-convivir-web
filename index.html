Módulos principales

Una plataforma especializada, no una planilla ni un módulo genérico.

Centraliza el ciclo operativo de convivencia escolar desde el primer registro hasta el cierre documentado.

01

Incidentes y anotaciones

Registro guiado de situaciones, contexto, participantes, clasificación inicial y acciones asociadas.

02

Alertas y derivaciones

Asignación de responsables, plazos, derivaciones internas y seguimiento de vencimientos.

03

Gestión de casos

Timeline, protocolos internos, evidencias, entrevistas, formularios y checklist de cierre.

04

Reportes ejecutivos

Indicadores por periodo, curso, tipo, estado y responsable para apoyar decisiones institucionales.

Flujo operacional

Del registro inicial al cierre documentado.

Reduce registros dispersos, pérdida de evidencias y seguimiento informal de situaciones sensibles. Cada acción queda asociada a estado, responsable y respaldo.

01
Registrar situaciónIncidente, anotación o evento relevante con datos mínimos y contexto institucional.
02
Evaluar y derivarClasificación, alerta, responsable, derivación interna y plazos de seguimiento.
03
Gestionar casoTimeline, evidencias, entrevistas, formularios, medidas internas y estados controlados.
04
Cerrar con trazabilidadChecklist, validación de acciones, documentos asociados y reapertura auditada si corresponde.

Arquitectura SaaS

Preparada para operación institucional segura.

Separación por cliente y establecimiento, control de permisos, auditoría y API institucional para integraciones controladas.

A

Aislamiento multi-tenant

Datos segregados por sostenedor, colegio y alcance efectivo del usuario.

R

Roles y permisos

Acceso diferenciado para dirección, convivencia, inspectoría, docentes y soporte.

L

Auditoría crítica

Registro de acciones sensibles, cambios de caso, descargas y uso de API.

API

API institucional

Conexión server-to-server con scopes, trazabilidad y control por cliente técnico.
